# MVP Test Plan

日期：2026-06-11  
来源：[MVP_IMPLEMENTATION_PLAN.md](./MVP_IMPLEMENTATION_PLAN.md)

## Critical Paths

1. 学员创建，手机号重复允许，身份证号重复拒绝。
2. 缴费草稿录入三个金额，后端正确计算实际收款。
3. 财务对照凭证重新输入金额后正式入账。
4. 正式缴费不可修改或删除，只能作废重录。
5. 退款必须关联缴费，累计退款不得超过实际收款。
6. 存在有效退款时缴费不可作废。
7. 作废退款后剩余可退金额恢复。
8. 附件仅授权用户可访问，正式入账后被冻结。
9. 所有正式入账和作废操作产生审计日志。
10. 备份可恢复数据库记录和附件。

## Required Test Suites

- 单元测试：金额、状态机、身份证标准化、权限判定。
- 集成测试：数据库约束、事务回滚、并发退款、审计日志、附件生命周期。
- E2E：学员、缴费、退款、作废重录和错误恢复完整流程。
- 安全测试：认证、授权、CSRF、附件访问、敏感数据与日志。
- 运维测试：迁移、部署、备份与恢复。

## Adversarial Test Matrix

| 场景 | 期望结果 |
|---|---|
| 两个退款确认同时提交 | 最多一笔成功，累计退款不超额 |
| 退款确认与缴费作废同时提交 | 只有符合锁内最新状态的操作成功 |
| 附件上传中尝试入账 | 入账被拒绝，草稿保持可恢复 |
| 数据库提交前对象存储失败 | 无正式记录产生，附件标记失败或待清理 |
| 数据库提交后清理任务失败 | 正式记录保持一致，扫描能发现待处理对象 |
| 同一幂等键重复请求 | 返回同一结果，不产生第二条记录 |
| 同一幂等键不同请求体 | 返回冲突，不执行写入 |
| 直接 SQL 修改正式业务字段 | 数据库拒绝 |
| 直接 SQL 更新或删除审计日志 | 数据库拒绝 |
| 金额为负数、超大值、三位小数 | 保存与入账均被拒绝 |
| 审计写入失败 | 整个业务事务回滚 |
| 学员信息入账后被编辑 | 正式账务仍显示原身份快照 |
| 备份恢复 | 附件哈希、密钥、账务汇总和审计链一致 |

## Release Gates

- 关键路径测试全部通过。
- 并发退款测试稳定通过。
- 正式记录修改与删除尝试全部被拒绝。
- 高危安全问题为零。
- 完成一次空环境恢复演练。
